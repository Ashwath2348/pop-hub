Programming Concepts: OOP vs POP

1. Introduction to Structures in C

The Need for Structures

Variables sometimes influence each otherâ€™s values. A change in one may necessitate a corresponding change in another. Structures help group related variables together to manage this situation more effectively.

        int d, m, y;  // day, month, year

Although these variables represent a single concept (a date), there is no mechanism to group them together in C. Consider a function next_day() to calculate the next day:

        void next_day(int *, int *, int *);

Without structures, passing these variables together in a group is difficult, which might lead to incorrect data being passed to the function. Structures solve this problem by grouping these variables together.

Why Does C Enable Us to Create Structures?

C allows creating user-defined types (structures) to group variables together under a single name. This enhances data organization and safety.

        struct date {
            int d, m, y;
        };

Now, a function like next_day() can take the address of the structure directly, avoiding mistakes in passing unrelated variables.

Limitations of Structures

  • C structures can't contain functions, only data members.
  • Data is not secure and can be modified by any function.
  • Associated functions are not part of the structure itself, making it difficult to maintain integrity in larger applications.

2. Issues of Procedure-Oriented Programming (POP) and Data Security

Issues of Procedure-Oriented Programming

  • Data Security: In POP, data is not protected from modification by external functions. Functions and data are separate, so any function can access and modify global data.
  • Accessing Data: Functions can share global data, leading to data being passed around freely and risking unintentional modifications.
  • Lack of Control: There's no strict mechanism to control access to data, which makes it difficult to ensure that data remains consistent and secure.

How Object-Oriented Programming Solves Data Security Issues

OOP solves this problem with the concept of data encapsulation. Data is hidden and can only be accessed through specific methods inside the object. The class controls access to its data, ensuring only authorized modifications.

3. What is Object-Oriented Programming (OOP)?

OOP is a programming paradigm that models real-world problems using objects. Objects are instances of classes and contain both data (attributes) and functions (methods). OOP helps organize code better and enhances reuse and modularity.

Salient Features of OOP and POP Comparison

Procedure-Oriented Programming (POP):

  • Focus on functions and procedures.
  • Programs are divided into functions that manipulate global data.
  • Follows a Top-Down approach in program design.

Object-Oriented Programming (OOP):

  • Focus on data and objects.
  • Programs are divided into objects that contain both data and methods.
  • Follows a Bottom-Up approach in program design.

4. Differences Between Procedure-Oriented and Object-Oriented Techniques

Feature Procedure-Oriented Programming (POP) Object-Oriented Programming (OOP)

Focus Functions and actions Data and objects

Data Security Less secure More secure (Data hiding)
Expansion Difficult to add new data and functions Easy to add new data and functions

5. Characteristics of Object-Oriented Programming (OOP)

i) Encapsulation

Encapsulation combines data and functions into a single unit called a class. Data is hidden and can only be accessed through public methods.

ii) Polymorphism

Polymorphism allows one function to behave differently based on the object calling it. This can include function overloading or operator overloading.

iii) Inheritance

Inheritance allows one class to inherit the properties and methods of another class, enabling reusability of code.
